## 简介

kubernetes是开源的linux容器自动化运维平台，可以自动化部署，扩展，管理容器应用。


## 安装 

kubernetes三种安装方式
* minikube：启动一个单点k8s，不能用于生产环境
* kubeadm：启动k8s集群
* 二进制：比较麻烦， 可用于生产环境



#### kubeadm 安装方式

关闭防火墙：

```
systemctl stop firewalld
systemctl disable firewalld
```

关闭selinux：
```
sed -i 's/enforcing/disabled/' /etc/selinux/config
setenforce 0
```

关闭swap：
```
swapoff -a # (临时)
vim /etc/fstab # (永久)  将 swap 那一行 注释掉即可
```

添加主机名与IP对应关系：

```
vim /etc/hosts
```

```
192.168.0.11 k8s-master
192.168.0.12 k8s-node1
192.168.0.13 k8s-node2
```


同步时间：
```
yum install ntpdate -y
ntpdate ntp.api.bz
```

保证每个节点有唯一主机名、mac地址、product_uuid：

```
ip link ：可以查看 mac地址 ，
cat /sys/class/dmi/id/product_uuid ：查看 product_uuid

vim /etc/hostname ：修改主机名， 重新进终端即可， 
hostname k8s-master  ：这个最好也执行一下，
```


安装 kubeadm kubectl kubelet：
```
cat << EOF > /etc/yum.repos.d/kubernetes.repo
[kubernetes]
name=Kubernetes
baseurl=https://mirrors.aliyun.com/kubernetes/yum/repos/kubernetes-el7-x86_64
enabled=1
gpgcheck=1
repo_gpgcheck=1
gpgkey=https://mirrors.aliyun.com/kubernetes/yum/doc/yum-key.gpg https://mirrors.aliyun.com/kubernetes/yum/doc/rpm-package-key.gpg
EOF
```

安装指定版本：
```
yum install -y kubelet-1.11.2 kubeadm-1.11.2  kubectl-1.11.2  --disableexcludes=kubernetes
```


```
systemctl enable kubelet && systemctl start kubelet
```

创建一个shell脚本：
```
vim sh1.sh
```

脚本内容：
```

ETCD_VERSION=3.2.18
DASHBOARD_VERSION=v1.8.3
FLANNEL_VERSION=v0.10.0-amd64

# 基本组件
docker pull registry.cn-hangzhou.aliyuncs.com/google_containers/kube-apiserver-amd64:$K8S_VERSION
docker pull registry.cn-hangzhou.aliyuncs.com/google_containers/kube-controller-manager-amd64:$K8S_VERSION
docker pull registry.cn-hangzhou.aliyuncs.com/google_containers/kube-scheduler-amd64:$K8S_VERSION
docker pull registry.cn-hangzhou.aliyuncs.com/google_containers/kube-proxy-amd64:$K8S_VERSION
docker pull registry.cn-hangzhou.aliyuncs.com/google_containers/etcd-amd64:$ETCD_VERSION

docker pull registry.cn-hangzhou.aliyuncs.com/google_containers/coredns:$DNS_VERSION

docker tag registry.cn-hangzhou.aliyuncs.com/google_containers/kube-controller-manager-amd64:$K8S_VERSION k8s.gcr.io/kube-controller-manager-amd64:$K8S_VERSION

docker tag registry.cn-hangzhou.aliyuncs.com/google_containers/coredns:$DNS_VERSION k8s.gcr.io/coredns:$DNS_VERSION
```

执行脚本：
```
bash sh1.sh
```

启动 k8s cluster：
```
kubeadm init --kubernetes-version=1.11.2 --pod-network-cidr=10.244.0.0/16 --apiserver-advertise-address=192.168.0.118
```
192.168.0.118 是k8s-master的ip


kubeadm init 显示的信息，包含下面三个命令，直接执行：
```
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

在master上执行命令，查看节点：
```
kubectl get node
```

kubeadm init 显示的信息 包含类似下面的命令， 在node上执行这个命令：
```
kubeadm join 192.168.0.118:6443 --token ig0ts3.3thd8vo5qtfky3r2 --discovery-token-ca-cert-hash sha256:e0bd26b2c8add45e3a26c69c471b323100cff32dd721f4a62e8bb9ff59acccee
```


安装网络插件：
```
kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/v0.10.0/Documentation/kube-flannel.yml
 
kubectl get pods  ：
kubectl get pods -n kube-system  ： 需要指定命名空间 
```

## 基本概念

#### Master
Master是集群中的控制节点

Master节点上有以下进程：
* Kubernetes API server：向外提供crud接口
* Kubernetes controller manager：所有资源对象的自动化控制中心
* Kubernetes scheduler：负责调度Pod

#### Node
除了Master，其他节点称为Node，Node负责工作负载

* kubelet：负责Pod对应的容器的创建、启动、停止。
* kube-proxy:实现Kubernetes Service的通信和负载均衡。
* Docker Engine


#### Pod

Pod：
* 根容器、Pause容器
* 普通容器

#### Label
Label是键值对，可以附加到Node、Pod、Service、RC上


#### Replication Controller (RC)
RC定义期望场景



#### Deployment
Deployment是RC的升级


#### Horizontal Pod Autoscaler (HPA)
自动对Pod进行横向扩容


#### Service
Service是一组Pod的逻辑抽象


#### Volume
Volume是Pod中能够被多个容器访问的共享目录


#### Persistent Volume
PV是集群中某个网络存储中对应的一块存储

#### Namespace
Namespace 命名空间，实现多租户


#### Annotation
Annotation类似Label
