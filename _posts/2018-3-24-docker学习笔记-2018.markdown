## docker

有没有遇到这些问题？

* 在同一个机器上部署两个程序，两个程序依赖同一个环境变量，所需要的环境变量的值还是不同的
* 多个程序要部署到一个机器上，各个程序依赖的语言的版本不一致
* 部署的时候需要安装依赖，在多个机器上安装的依赖不一致，导致问题
* 程序运行结束后，想把依赖的文件删除掉
* 一个进程被入侵，从这个进程可以访问到文件，可以看到其他进程的信息

有了docker，上面的问题可以优雅解决。将程序和依赖全部打包到一个docker镜像里面，在机器上根据docker镜像启动容器即可。

docker使用命名空间和cgroup实现进程隔离，资源隔离


## docker安装 (CentOS)

https://docs.docker-cn.com/engine/installation/linux/docker-ce/centos/

```
sudo yum remove docker \
                docker-common \
                docker-selinux \
                docker-engine            
                  
sudo yum install -y yum-utils \
                    device-mapper-persistent-data \
                    lvm2

sudo yum-config-manager \
    --add-repo \
    https://download.docker.com/linux/centos/docker-ce.repo  
 
sudo yum localinstall -y https://download.docker.com/linux/centos/7/x86_64/stable/Packages/docker-ce-selinux-17.03.3.ce-1.el7.noarch.rpm
  
sudo yum install -y docker-ce-17.03.3.ce 
 
sudo systemctl enable docker && systemctl start docker
```

## docker命令


#### image

```
docker images 
docker image ls
docker image pull nginx:1.11
docker image rm nginx:1.11
docker image tag nginx:1.12 nginx:v1
```

#### container

```
docker container ls 
docker container ls -a
docker container ls -aq

docker container rm <hash>      

docker rm $(docker ps -q )：删除已经停止的容器
docker rm -f $(docker ps -aq )：删除所有容器
docker container rm $(docker container ls -a -q)      

docker start name
docker stop name
docker rm name
docker container kill <hash>         

docker container stop $(docker container ls -a -q) 
```

#### exec 

```
docker exec -it nginx1 bash
docker exec -it nginx1 sh 
```


## 容器管理
当容器数量较少时，可以手动管理各个容器，但是当容器数量增多时，需要一些自动化工具

* docker-compose ： 单机情况下，一次启动多个容器
* docker swarm 
* kubernetes (k8s)
* lightweight kubernetes (k3s): 



## docker-compose 

https://docs.docker.com/compose/install/#install-compose

```
sudo curl -L "https://github.com/docker/compose/releases/download/1.24.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```


```
docker-compose up
```
